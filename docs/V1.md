# ATLAS / Hermes v1

v1 is the first release where the organism has one executable truth surface.
It is not a claim of unconstrained autonomy. It is a local, operator-owned
system that can remember, act, test, recover, grade claims independently, and
report degradation without inventing success.

## Identity and inference

- Hermes is the complete local organism.
- ATLAS is Hermes's executive and conversational surface.
- Station is the spine and append-only evidence notary.
- The executive route uses the authenticated Codex CLI subscription with
  `gpt-5.6-luna`. No OpenAI API key is required.
- Bounded read, build, crystallization, and fleet tasks may use other assigned
  Codex-family routes according to their failure surface.

## Release contract

The machine-readable contract is `v1.contract.json`. A v1 build must prove:

1. Direct conversation survives autonomous and system traffic.
2. Autonomy continues until its deadline or operator return and discovers new
   work instead of treating an idle tick as completion.
3. Memory compression is loss-aware; shards repair exact bytes and crystals
   remain pointers to evidence rather than substitutes for it.
4. External wings launch, exchange commands, surface felt state, and stop
   cleanly.
5. Claims remain unverified until graded by a path independent of their
   generator.
6. The mission path preserves certified, rejected, and honest-null outcomes.
7. The UI identifies sidecar generations and cannot present stale work as live.
8. The complete acceptance and organism-rehearsal surfaces pass.
9. The release commit is clean and bound by the `v1.0.0` tag.

Each self-improvement receipt is admitted through `npm run spiral:seal`. A
subsystem/capability pair or evidence set already present in the recent ledger
cannot be counted again. Further work on that vector is recorded explicitly as
a continuation and does not increase the new-vector count.

## Operator commands

```text
npm start
npm run v1:status
npm run v1:verify
npm run v1:release
```

`v1:status` checks the contract, required organs, provider route, exact
orchestrator assignment, and operator documentation. `v1:verify` executes the
full local acceptance suite and whole-organism rehearsal and writes the
candidate receipt. `v1:release` adds the clean-tree gate and is the final
pre-tag command.

## Safety boundary

v1 autonomy is local and time-bounded. It may improve files and run local tools
inside the operator-authorized scope. It does not independently publish,
purchase, contact third parties, deploy externally, or reinterpret a hypothesis
as a verified claim.
